Terms used in OOP

  • Abstraction: identifying shared characteristics of our work (methods or procedures) and the object itself
  • Object: A “class” defines the common traits of similar tings, such as mammals ... and then you might have child classes such as dog, cat, dolphin and the like that share (a) the base (or parent) class properties and (b) exhibit their own unique characteristics ...
  • Class: is just the object when expressed in code.
  • Objects/Classes exhibit 3 features that keep the object's properties and what the object does (aka methods) in the same block of code (the class):
    1. Encapsulation: Encapsulation is wrapping the data under a single, consolidated unit. In OOP, it is defined as binding data with a function that manipulates it.
    2. Inheritance: Inheritance is the ability of one class to derive its characteristics from another class.
    3. Polymorphism : Polymorphism refers to a programming languageâ€™s ability to process objects uniquely according to their data type and/or class.   • Interface (API): is usually a module for a specific communication between other applications and languages, such as the mysql.connection interface [you may already be familiar with JDBC:ODBC, or beautifulsoup, and other APIs].
